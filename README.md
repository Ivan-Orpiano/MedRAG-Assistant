MediRAG is a Retrieval-Augmented Generation (RAG) system that lets healthcare professionals and researchers upload clinical guidelines, research papers, hospital SOPs, and treatment protocols, then ask natural-language questions and receive answers **grounded strictly in the uploaded knowledge base** — complete with source citations, page numbers, and excerpts.

---


## Key Features

### 📄 Document Management
- Upload clinical guidelines, research papers, SOPs, and treatment protocols in **PDF, DOCX, and TXT** formats
- Automatic text extraction, cleaning, and preprocessing
- **OCR support** for scanned/image-based PDFs
- Document categorization, versioning, update, and deletion
- Background ingestion pipeline (non-blocking uploads)

### 🔍 Retrieval-Augmented Generation (RAG)
- Optimized chunking strategy tuned for medical text
- Embeddings generated via **OpenAI `text-embedding-3-large`**
- Vector storage in **Qdrant** (preferred) or **Pinecone**
- **Hybrid search** — combines semantic (vector) search with keyword search for higher recall
- **Metadata filtering** (document type, category, date, author, version)

### 💬 Grounded Conversational AI
- MediRAG answers **only** from retrieved context — no fabrication, no external knowledge
- Explicit "information not available in the knowledge base" fallback when evidence is insufficient
- Every response includes **citations**: source document, title, page number (when available), and relevant excerpt
- Multi-turn **conversation history** while keeping each answer grounded in retrieved evidence
- **Streaming responses** for real-time token-by-token output

### 🔐 Access Control & Administration
- JWT-based **authentication**
- **Role-Based Access Control (RBAC)**: `Administrator`, `Doctor`, `Researcher`
- Admin dashboard for monitoring document indexing status and system usage
- Full **chat history** per user

### ⚙️ Platform & Reliability
- **Celery** background workers for ingestion, embedding, and OCR jobs
- **Redis** caching for embeddings/query results and as the Celery broker
- **API rate limiting**
- Centralized structured **logging**
- Comprehensive **error handling**
- **OpenAPI/Swagger** documentation auto-generated by FastAPI
- **Unit tests** across ingestion, retrieval, and chat modules
- Fully **Dockerized** deployment

## How MediRAG Works

1. **Upload** — A Doctor, Researcher, or Administrator uploads a PDF, DOCX, or TXT document through the frontend.
2. **Ingestion (async)** — A Celery task extracts text (with OCR fallback for scanned PDFs), cleans and normalizes it, and splits it into optimized, overlapping chunks.
3. **Embedding** — Each chunk is embedded using OpenAI `text-embedding-3-large` and stored in Qdrant/Pinecone along with metadata (document title, page number, category, version).
4. **Query** — A user asks MediRAG a question. The retrieval module performs hybrid search (vector similarity + keyword match), optionally filtered by metadata, to fetch the most relevant chunks.
5. **Grounded generation** — Retrieved chunks are passed as context to the OpenAI GPT model with a strict system prompt instructing MediRAG to answer **only** from the provided context and to explicitly state when the knowledge base lacks sufficient information.
6. **Response** — The answer streams back to the user with inline citations (document title, page number, excerpt) and is saved to conversation history.

## Security Considerations

- Passwords hashed with a strong algorithm (e.g., bcrypt/argon2)
- JWT access tokens with configurable expiry
- Role-based route protection enforced at the API layer
- File type and size validation on upload
- Rate limiting to mitigate abuse
- No PHI (Protected Health Information) should be uploaded — MediRAG is designed for **reference documents only** (guidelines, protocols, research), not patient records
